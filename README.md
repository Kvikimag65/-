# Описание программного проекта
В основе проекта лежит десктопное приложение для управления парком транспортных средств. Главная цель — показать на практике разделение логики и интерфейса, а также использование базовых принципов ООП.
# Технологический стек
GUI-фреймворк: Я выбрал PyQt6, так как он дает полный контроль над интерфейсом и имеет удобную систему сигналов/слотов. Хранение данных: Использую формат JSON. Это позволяет легко сохранять текущее состояние объектов и считывать их при перезапуске приложения.
# Структура и логика 
классов Для чистоты кода я разделил функционал на три ключевых блока:1. Модель данных (Класс RoyalCarriage) Это описание самой сущности транспорта. Здесь я реализовал автоматическую генерацию ID и хранение параметров (тип, скорость, статус). Также добавил метод to_dict, который подготавливает данные для сохранения в файл.2. Логика и управление (Класс KingdomRegistry) Этот класс работает как «мозг» программы. Он хранит список всех объектов и отвечает за вычисления: добавление новых записей и расчет статистики (минимумы, максимумы, группировка по статусам). Важно, что этот класс никак не связан с кнопками или окнами — только чистая работа с данными.3. Визуальная часть (Класс ThroneRoom) Основное окно приложения на базе QMainWindow. В нем я настроил:Layouts: Использовал вертикальные и горизонтальные контейнеры, чтобы интерфейс не «разваливался» при изменении размеров окна.События: Все действия (клики по кнопкам, выбор в списке) привязаны к методам через .connect().Работа с файлами: Для открытия и сохранения базы я задействовал стандартные диалоговые окна QFileDialog.
# Реализованные возможности
Учет объектов: Создание записей и их визуализация в списке.Динамическое управление: Быстрая смена статуса транспорта через кнопки управления.Аналитика: Система собирает данные по всему списку и выводит отчет через всплывающее окно.Персистентность: Данные не теряются после закрытия программы благодаря импорту и экспорту в JSON
