# 1. Общее описание проекта
Проект представляет собой десктопное приложение с графическим пользовательским интерфейсом (GUI), предназначенное для учета и управления состояниями объектов (транспортных средств). Программа реализует базовые принципы объектно-ориентированного программирования (ООП) и демонстрирует четкое разделение бизнес-логики и интерфейса пользователя.

# 2. Используемые технологии

Язык программирования: Python 3.

Графический фреймворк: PyQt6 (выбран за высокую производительность, гибкость компоновки и удобную событийно-ориентированную модель).

Хранение данных: Встроенный модуль json для персистентности (сохранения) состояния приложения между сессиями.

# 3. Архитектура и основные компоненты (Классы)
Код логически разделен на три основных класса, что упрощает масштабирование и поддержку:

Класс RoyalCarriage (Модель данных / Сущность)
Представляет конкретный объект предметной области. Он инкапсулирует в себе атрибуты отдельного транспорта: уникальный идентификатор (генерируется автоматически), тип/породу, скорость и текущий статус (например, «в движении» или «остановлен»). Содержит метод to_dict() для подготовки объекта к сериализации.

Класс KingdomRegistry (Контроллер данных / Реестр)
Отвечает за хранение коллекции всех объектов (список fleet) и управление ими. В этом классе сосредоточена вся бизнес-логика программы: добавление новых записей, вычисление агрегированной статистики (поиск минимальной/максимальной скорости, подсчет статусов) и логика импорта/экспорта данных в JSON-файл. Этот класс абсолютно независим от графического интерфейса.

Класс ThroneRoom (Графический интерфейс)
Наследуется от QMainWindow из библиотеки PyQt6. Отвечает исключительно за отображение данных и сбор пользовательского ввода.

Компоновка (Layouts): Интерфейс построен с использованием менеджеров геометрии QVBoxLayout (вертикальное размещение) и QHBoxLayout (горизонтальное размещение), что делает окно адаптивным.

Взаимодействие (Signals and Slots): Реализована событийно-ориентированная архитектура. Действия пользователя (нажатие кнопок QPushButton, выбор элемента в QListWidget) генерируют сигналы, которые связаны с методами-обработчиками (слотами) через механизм .connect().

Работа с файлами: Для удобства пользователя импорт и экспорт базы данных осуществляется через системные диалоговые окна QFileDialog, что позволяет динамически выбирать пути сохранения файлов.

# 4. Главный функционал

CRUD-операции (частично): Добавление новых элементов в систему, чтение списка на экране.

Управление состояниями: Изменение статуса выбранного объекта по нажатию кнопок управления.

Агрегация данных: Сбор и вывод статистического отчета по всей коллекции объектов через QMessageBox.

Сериализация: Сохранение состояния объектов в человекочитаемый формат JSON и обратная десериализация (восстановление) при загрузке.
